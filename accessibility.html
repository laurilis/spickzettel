Digital Accessibility

make Websites accessible for people with disabilities!

  • Screen readers that parse a website for a user with visual impairments
  • Videos on websites are closed-captioned for individuals with hearing impairments
  • Images include “alt text” for individuals with visual impairments
  • Websites must be navigable by keyboard for
  • users who may not be able to operate a mouse (i.e., navigating using the “Tab” on a keyboard)
Web Accessibility Initiative hat Guidelines veröffentlicht namens ARIA (Accessible rich Internet Applications). Writing more semantic HTML tags and image captions also can improve your website’s standing in search engines, as they provide the search engine more information about the content that your site delivers.


Best Practices

  • Kontrast & Colors: auf Kontrast achten & color impairment beachten (es gibt websites zur Vorschau wie toptal)
  • mehr Reichweite: As an added benefit, having a keyword in an h1 tag will typically impact search results more than having the same keyword in a p tag
  • h1 nur einmal pro Seite best practice, h2 for each sibling section
  • Semantic HTML: The quickest way of improving accessibility for screen readers is to use the appropriate tags for a given task. Bsp: HEADER sollte navigationsleiste, suchleisten, introductionary elements & links enthalten
  • role attribute - für screen reader, bsp. role="complementary" attribute and value can help a screen reader user understand that the information in the code editor is complementary (or supporting) to the information you are reading right now. It helps users of screen readers identify the relationship between the two sections of the page. zB role=”presentation” —> inhalt kann geskipped werden, zb ol oder ul
    —> https://www.w3.org/TR/html-aria/#allowed-aria-roles-states-and-properties
  • The property aria-label gives the screen reader additional information to read out loud to the user. bsp <p aria-label="Artist">Armand Cabrera, 2010</>
  • ALT ATTRIBUTE works like aria-label but has additional functionality. Alt is used to describe an image (or several other elements). Aria will in contrast never be displayed. If an image is described by text near the image, do not duplicate the description in the alt attribute. Use an empty alt attribute instead.
