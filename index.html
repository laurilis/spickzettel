Tabellen

  • Struktur <table> umschließt <thead>, <tbody> und ggflls <tfoot>
  • neue Zeile, auch für Leitkategorien beginnen mit <tr>
  • Leitkategorie unter <thead> und <tr> als <th scope=”col”>erste Spalte</th> - je mehr Elemente, desto mehr Spalten
  • col = Spalte row = Zeile
  • Zelleninhalt innerhalb von <tbody> in <tr> als <td>
  • Zelleninhalt über col oder row strecken, Zahl ist Zellenzahl: <td colspan="2">Zelleninhalt</td>
  • <tfoot> benötigt wohl keine extra <tr>

Meta Tag

Based on the size of the viewport, the meta tag <meta> is used to instruct the browser on how to render the webpage’s scale and dimensions. If a web page is 960px and the device is 320px wide. Adding the viewport meta tag will squish the content down until it is 320px

  •   <meta name="viewport" content="width=device-width, initial-scale=1">
  • name="viewport" attribute: tells the browser to display the web page at the same width as its screen
  • content attribute: defines the values for the <meta> tag, including width and initial-scale
  • width=device-width key-value pair: controls the size of the viewport in which it sets the width of the viewport to equal the width of the device
  • initial-scale=1 key-value pair: sets the initial zoom level (Read more about scale at MDN’s viewport documentation)

<form action="/example.html" method="POST"> </form>
The action attribute determines where the information is sent. The method attribute is assigned a HTTP verb that is included in the HTTP request.
Form elements can contain children like header or p. Within form element we need:

  • <input type="text" name="first-text-field"> - When the form containing input “important details” is submitted, the text: ` "first-text-field=important details"` is sent to `/example.html` because the value of the `name` attribute is `"first-text-field"` and the value of `value` is `"important details"`. —> default value prefilled upon value=”prefilled message”
  • LABEL: For a user to properly identify an <input> we use the appropriately named <label> element. <Input> needs ID and <label> needs FOR attribute so they go together.
    <form action="/example.html" method="POST">
       <label for="meal">What do you want to eat? </label>   
       <input type="text" name="food" id="meal"> </form>

Types of Fields

  • Passwords - An <input type ="password"> element will replace input text with another character like an asterisk (*) or a dot (•)
  • Numbers - step attribute which creates arrows inside the input field to increase or decrease by the value of the step attribute, see   <input id="years" name="years" type="number" step="1"> -> step 1 = ganze zahl +-
  • Range Input - type=”range” —> <input id="volume" name="volume" type="range" min="0" max="100" step="1"> will create a slider
  • Checkbox Input - <label for="cheese">Extra cheese   <input id="cheese" name="topping" type="checkbox" value="cheese">
  • Radio Button - present multiple options & only allow for one selection, like terms&conditions, see
    <form>   <p>What is sum of 1 + 1?</p>
      <input type="radio" id="two" name="answer" value="2">
      <label for="two">2</label>   
      <input type="radio" id="eleven" name="answer" value="11">
      <label for="eleven">11</label> </form>
  • DropDown List - <form>   <label for="lunch">What's for lunch?</label>
      <select id="lunch" name="lunch">
        <option value="pizza">Pizza</option>
        <option value="curry">Curry</option>
        <option value="salad">Salad</option>   </select> </form>
  • Datalist - used with an <input type="text"> element. The <input> creates a text field that users can type into and filter options from the <datalist>. Difference select & datalist: If none of the options match, the user can still use what they typed in. (datalist).
    <form>
      <label for="city">Ideal city to visit?</label>
      <input type="text" list="cities" id="city" name="city">
      <datalist id="cities">
        <option value="New York City"></option>
        <option value="Tokyo"></option>
        <option value="Barcelona"></option>
        <option value="Mexico City"></option>      </datalist> </form>
  • Textarea - if more information needed, then <textarea> instead of <input>. The value of <textarea> is the text written inside he box. If we want to add a default value to <textarea> we would include it within the opening and closing tags like so: <textarea>Adding default text</textarea>
    <form>
      <label for="blog">New Blog Post: </label>   
      <textarea id="blog" name="blog" rows="5" cols="30">
      </textarea> </form>
  • Submit button
    <form>
      <input type="submit" value="Send" > </form> -> button will say "Send", if no value attribute, then will display "Submit"
  • Requiring Input - <input id="allergies" name="allergies" type="text" required>
    number - type=”number” min=”1” max=”5”
    characters - minlength="5" maxlength="250"
    following pattern (regex) - pattern=”[0-9]{14,16}" which checks that the user provided only numbers and that they entered at least 14 digits and at most 16 digits.
